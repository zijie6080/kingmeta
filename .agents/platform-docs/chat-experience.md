# chat-experience

> Chat composer, file attachments, Drive imports, suggestions, live browser, and message controls

The editor chat surface is more than a text box. Users can send text, stop a running response, attach local files, import files from Google Drive when that entry point is visible, refresh and use prompt suggestions, and dictate short input through speech-to-text. When live voice is available and the composer is empty, a call button starts a real-time voice session; if the button is hidden, say live voice is not currently available rather than inventing a manual endpoint.

The message list supports loading older messages, deleting messages, showing grouped tool activity, submitting interactive tool approvals or follow-up input, reading assistant responses aloud when audio playback is available, and opening feedback from the sidebar or message feedback entry points. Queued or pending user messages can be edited or deleted before the run consumes them. When a managed browser session is active, the composer can show a live browser control so the user can keep the browser visible while chatting.
